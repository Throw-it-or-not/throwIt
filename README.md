## 🔎 팀 개발 플로우 요약

## 🌵 Git 브랜치 전략

```jsx
| * refector
| * style
| * bug
| * fix
| * feature
|/
* develop
|
* main

# 브랜치 네이밍
# 브랜치/이슈번호-작업요약-이니셜
# ex) feature/19-changeFishImg-kdh 
```

## 📁 폴더 구조

```jsx
HTML ─ index.html      # HTML 구조

/CSS
├── common.css         # 전체 스타일 초기화 설정
└── main.css           # 각 기능별 다자인 스타일 

/JS
  ├── app.js           # 최상위 진입점
  ├── dom/             # dom 분리 폴더
  └── modules/         # 모듈 분리 폴더
     ├── catch/        # 낚시 모달 기능 모듈
     ├── fish/         # 바다 스테이지 기능 모듈
     └── menu/         # 메인 메뉴 기능 모듈
  
/images                # 아이콘 및 이미지 등
```

## 🛠 협업 방식

- 매일 회의를 통해 당일 작업 이슈와 To Do 리스트 정리
- 각자 작업 브랜치를 만들어 작업 후 PR로 병합
- 브랜치명 및 커밋 메시지는 팀장이 정한 규칙에 따라 작성
- 코드 리뷰는 GitHub PR을 통해 최소 1명 이상 리뷰 후 머지

## ▶️ 작업 순서

- 브랜치 생성: git swtich -c [브랜치/이슈번호-작업요약-이니셜]
- 기능 구현 & 커밋: git commit -m “feat: #이슈번호 기능  설명”
- 원격 저장소 push: git push origin [로컬 브랜치]
- Rull Request: 풀 리퀘스트 작성
- develop 브랜치 병합: 팀장 or 팀원 테스트, 리뷰 후 develop 브랜치에 머지
- 병합 후 브랜치 삭제

## ✏️ 커밋 메시지 규칙

| 태그 | 의미 |
| --- | --- |
| `feat` | 기능 추가 |
| `fix` | 버그 수정 |
| `style` | 스타일/CSS 수정 |
| `refactor` | 리팩토링 |
| `docs` | 문서/주석 수정 |
| `chore` | 설정/빌드 기타 작업 |
| `test` | 테스트 코드 추가/수정 |

## 🕸️ GitHub 기반 협업 진행

1. **문제 인식**: 낚시 게임 진행 중 성공 범위가 아님에도 게이지 바의 색상이 업데이트 되지 않는 문제 발생
2. **이슈 생성**: 깃허브에서 이슈 생성

![image](https://github.com/user-attachments/assets/dbb1a6dd-85fa-4792-be80-3f43bbae56aa)


1. **브랜치 작업**:  develop 브랜치에서 새 브랜치 생성, 이동 후 작업

```jsx
# develop 브랜치에서 새 브랜치 생성 및 이동
git checkout develop
git pull origin develop
git branch bug/32-fishingGauge-kgj
git switch bug/32-fishingGauge-kgj
```

1. **작업 업 후 커밋**

```jsx
# 작업 후 커밋
git add .
git commit -m "fix: 게임 종료 시 게이지 업데이트, updateGaugeColor 파라미터 부족 해결"
```

1. **원격 저장소로 push**

```jsx
# 원격 저장소로 push
git push origin bug/32-fishingGauge-kgj
```

1. **Pull Request 요청**
    
    ![image 1](https://github.com/user-attachments/assets/e538ba89-873e-4a29-b240-2935e0303cf9)

    

1. **테스트**

원격 브랜치를 로컬로 가져옴

```jsx
# 원격 브랜치를 로컬로 가져옴
git fetch origin bug/32-fishingGauge-kgj
git switch bug/32-fishingGauge-kgj
```

Pull Request 에서 변경된 내용을 사용자의 이용 시나리오에 맞춰 Case를 나눠서 확인

- 게임을 시작하는 순간 게이지의 색상이 제대로 변경되는지 확인
- 게임을 동작하는 와중에 게이지의 색상이 성공 범위(70%~90%)에 따라 색상이 변경되는지 확인
- 게임이 끝날 때 게이지의 색상이 제대로 반영되는지 확인

테스트 완료 후 문제가 없으면 코멘트 작성 후 develop 브랜치에 Merge

![image 2](https://github.com/user-attachments/assets/fcdedf20-9c45-47f6-8dee-7b7b54a5111f)


Merge 후 브랜치 정리

```jsx
git switch develop
git pull origin develop
git branch -d bug/32-fishingGauge-kgj
git push origin --delete bug/32-fishingGauge-kgj
```
