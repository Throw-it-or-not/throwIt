## 🌵 Git 브랜치 전략

```jsx
| * refector
| * style
| * fix
| * feature
|/
* develop
|
* main

# 브랜치 네이밍
# 브랜치명/이슈번호-작업요약-이니셜
# ex) feature/19-changeFishImg-kdh
```

## 📁 폴더 구조

```jsx
HTML ─ index.html      # HTML 구조

/CSS
├── common.css         # 전체 스타일 초기화 설정
└── main.css           # 각 기능별 다자인 스타일 

/JS
  ├── app.js           # 최상위 진입점
  ├── dom/             # dom 분리 폴더
  └── modules/         # 모듈 분리 폴더
     ├── catch/        # 낚시 모달 기능 모듈
     ├── fish/         # 바다 스테이지 기능 모듈
     └── menu/         # 메인 메뉴 기능 모듈
  
/img
  └── images/          # 아이콘 및 이미지 등
```

## 🛠 협업 방식

- 매일 회의를 통해 당일 작업 이슈와 To Do 리스트 정리
- 각자 작업 브랜치를 만들어 작업 후 PR로 병합
- 브랜치명 및 커밋 메시지는 팀장이 정한 규칙에 따라 작성
- 코드 리뷰는 GitHub PR을 통해 최소 1명 이상 리뷰 후 머지

## ✏️ 커밋 메시지 규칙

| 태그 | 의미 |
| --- | --- |
| `feat` | 기능 추가 |
| `fix` | 버그 수정 |
| `style` | 스타일/CSS 수정 |
| `refactor` | 리팩토링 |
| `docs` | 문서/주석 수정 |
| `chore` | 설정/빌드 기타 작업 |
| `test` | 테스트 코드 추가/수정 |
